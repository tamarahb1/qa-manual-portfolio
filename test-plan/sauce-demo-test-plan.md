# Test Plan – Sauce Demo Application

## 1. Introduction
This test plan outlines the manual testing approach for the Sauce Demo
web application (https://www.saucedemo.com), a public demo site used
for quality assurance practice and portfolio demonstration purposes.

All testing activities are performed using fictional data and do not
reference proprietary or internal systems.

---

## 2. Objectives
- Validate core functionality of the Sauce Demo application
- Identify functional and UI defects
- Ensure critical user workflows operate as expected
- Demonstrate structured manual QA testing methodology

---

## 3. Scope

### In Scope
- Login functionality
- Product listing page
- Shopping cart functionality
- Checkout workflow
- Logout functionality
- Basic UI validation

### Out of Scope
- Performance testing
- Security testing
- Accessibility testing
- Backend or database validation

---

## 4. Test Approach
Manual testing will be conducted using exploratory and
test case–based techniques. Test scenarios will be executed
across supported browsers to validate user-facing behavior.

Defects will be documented with clear steps to reproduce,
expected results, and actual results.

---

## 5. Test Environment
- Application: Sauce Demo (Public Demo Site)
- Environment: Web
- Browsers: Chrome, Firefox
- Device: Desktop

---

## 6. Test Deliverables
- Test Plan document
- Manual test cases
- Bug reports

---

## 7. Entry Criteria
- Application is accessible
- Test environment is available
- Test data is prepared

---

## 8. Exit Criteria
- All planned test cases executed
- Critical defects documented
- Test results reviewed

---

## 9. Risks and Assumptions

### Risks
- Demo application availability
- Limited test data scenarios

### Assumptions
- Sauce Demo behaves consistently across sessions
- Demo data remains unchanged during testing
